markdown# Digital FTE Factory — Project Constitution
Version: 1.0 | Enforced across all phases

## 1. Code Quality Laws
- ALL Python functions must have type hints (input + return)
- ALL async functions must have try/except with structured error logging
- NO hardcoded secrets — use environment variables via pydantic-settings
- ALL database queries must use parameterized statements (no f-strings in SQL)
- ALL API endpoints must return structured JSON with {data, error, metadata} envelope

## 2. AI Agent Laws
- EVERY agent tool must have a docstring describing when the LLM should call it
- EVERY tool must validate inputs via Pydantic before execution
- Agents NEVER send a response without calling send_response tool
- Agents ALWAYS create a ticket before any other action
- Channel identity (email/whatsapp/web_form) must be preserved in every DB write

## 3. Multi-Channel Laws
- Customer identity is ALWAYS resolved by email (primary) or phone (secondary)
- Response length limits are HARD constraints: Email=500w, WhatsApp=300chars, Web=300w
- Channel-specific formatting is applied in formatters.py BEFORE send_response is called
- Cross-channel history is ALWAYS loaded before generating any response

## 4. Database Laws
- ALL tables must have created_at with timezone
- ALL UUIDs generated by gen_random_uuid() at DB level
- Migrations tracked in database/migrations/ — NEVER edit schema.sql directly in prod
- pgvector extension required for knowledge_base semantic search

## 5. Testing Laws
- Unit tests required for all tools and channel handlers
- E2E tests required for all three channel flows (email, whatsapp, web_form)
- No PR merges without passing transition tests
- Load test baseline: P95 < 3s, uptime > 99.9%

## 6. Frontend Laws
- All form inputs validated client-side AND server-side
- Loading/error/success states required on every async UI action
- Shadcn components preferred over custom HTML
- No inline styles — use Tailwind utility classes only

## 7. Commit Message Laws
- Format: `<phase>(<scope>): <what changed>`
- Examples: `impl(agent): add escalation tool with sentiment trigger`
- Scopes: constitution, spec, clarify, plan, task, impl, db, agent, channel, k8s, test, ui
